# Заключительная итерация: 50 глубоких анализов + ITER11 (стадия снапшотов)

Дата: 2026-07-14. Решение о составе итерации принято агентом по запросу «реши сам»:
сначала довести прерванный финальный аудит из 50 анализов, затем сразу реализовать
одобренную идею «доворот в канон + полная таблица landmarks + dense-геометрия одним
снапшотом» как ITER11.

---

## Часть 1. Финальный аудит: 50 глубоких анализов

Раннер: `tests/audit_50_new_version.py` покрывал структурные/юнит-инварианты (60 проверок).
Новый раннер `tests/audit_final_50.py` покрывает сквозные и межмодульные инварианты:

- **Alignment (1–7)**: Umeyama identity/восстановление поворота/блокировка и восстановление
  масштаба, ортонормальность Эйлер-матриц, канонические углы всех 9 ракурсов против
  `CANONICAL_YAW_BY_VIEW_GROUP`, align_meshes_shared.
- **Compare-примитивы (8–15)**: shared-индексы, геодезическая дистанция поз (0° и 90°),
  косинусная дистанция alpha_id, монотонность ошибки при росте деформации.
- **Compare e2e на мэше BFM-размера 35709 (16–18)**: одинаковый мэш → error ~0, score >0.999;
  деформация 0.4 → ошибка на порядки выше; несовпадение размеров → fail-closed статус.
- **Калибровка (19–27)**: MAD, effective sample size, block-bootstrap CI, SNR-пол,
  quality bands, health status, персональные базлайны, нормализация ключей ячеек,
  NoiseModel (zone/cell-профили, рост предсказанного шума с pose-дистанцией).
- **Кэш (28–30)**: roundtrip массивов, эвикция старых записей, чувствительность хеша к dtype.
- **Вердикт (31–37)**: fail-closed на пустых уликах, сумма вероятностей = 1, сильный SNR →
  P(H2)>P(H0), монотонность правдоподобий по SNR, неинформативность UNAVAILABLE,
  устойчивость к нулевым правдоподобиям (без NaN), INSUFFICIENT_DATA-метка.
- **Отчёт (38–40)**: save/load roundtrip + стабильный fingerprint, acceptance режет
  постериоры с суммой ≠ 1, сериализация numpy-типов.
- **Каталог × policy (41–45)**: все `views` каждой из 1251 строк CSV парсятся в известные
  бакеты; специи бакетов ⊆ полного реестра и проходят `spec_allowed_for_bucket`;
  в left_profile нет утечек сторон R/B; выбранные имена всех 9 бакетов существуют в каталоге;
  runtime-гейт блокирует билатеральные метрики при yaw > 45°.
- **Legacy-runner e2e (46–47)**: синтетический контекст 35709 вершин → ≥100 конечных значений;
  пустой selection-конфиг сохраняет все значения.
- **Зоны + видимость (48–50)**: идентичные облака → зонные ошибки ~0 и валидный bone-summary;
  канthus-точки из орбиты; yaw-фейд затуханий на 70° и паритет util ↔ uv_module по
  видимости треугольников.

**Результат: 50/50 PASS с первого прогона, дефектов не выявлено.**
Лог: `/data/review_3ddfa/audit_final_50.json`.
Это ожидаемо: предыдущая итерация (60 проверок) выявила и исправила 12 классов дефектов.

## Часть 2. ITER11 — стадия снапшотов («extract once, compute later»)

Идея пользователя: не считать геометрические метрики на раннем этапе, а доворачивать модель
в канон и сохранять полную таблицу landmarks, чтобы все таблицы (калибровочный датасет и
основной анализ) получались из одних данных. Реализовано с поправкой: сохраняем не только
landmarks, но и dense-геометрию, иначе 70%+ каталога (F_zone/F_pair_zone/F9) невосстановимо.

### Новые файлы

- `util/snapshot.py` — `PhotoSnapshot`, `CanonTransform`, `canon_transform_from_pose`,
  `save_snapshot` / `load_snapshot` (npz, без pickle, атомарная запись),
  `to_metric_context` (мост в legacy-runner), `landmarks_table`.
- `tests/test_iter11_snapshot.py` — 10 тестов.

### Ключевые решения схемы `snapshot_v1`

1. **Канон — это трансформация, а не результат.** Хранится (R, t, scale); raw-вершины —
   источник истины; canon-пространство деривируется на лету. Смена определения канона
   не требует повторного 3DDFA-прогона (проверено тестом 03).
2. **Состав**: vertices_raw float32 (35709×3), triangles, normals_raw, alpha_id (80),
   exp_params (64), landmarks_106_raw (полная таблица; canon деривируется), visibility
   weights, quality-гейт, extras (focal=1015, source_schema=3ddfa_v3_iter1_v1).
3. **Fail-closed**: несовпадение schema_version, NaN в вершинах, out-of-range треугольники,
   неортонормальный поворот, кривые веса — всё отклоняется на входе/загрузке.
4. **Бюджет**: полный снапшот < 3 МБ (тест 10), фактически ~1.5 МБ compressed.
5. **Детерминизм**: два прогона legacy-runner по одному снапшоту дают идентичные значения
   метрик (тест 07) — гарантия «калибровка и основной анализ считаются одним кодом по одним
   данным».

### Тесты ITER11 (10/10 PASS)

roundtrip; соответствие канонизации `alignment` (undo pose → канонические углы бакета);
канон не запечён; нормали остаются единичными; таблица landmarks в двух пространствах;
снапшот → MetricContext → ≥100 конечных метрик; детерминизм; fail-closed по схеме;
валидация мусорных входов; бюджет размера.
